Network

Functional Storage is a modern, Titanium-based take on item drawers: clean single-item bulk storage where every slot holds one item type with a live count on the face, framed in wood through netherite for capacity tiers, linked into one network by a controller, and reachable through a searchable storage terminal for AE-lite access. If you mass-produce ingots, dusts, drops, or building blocks, this is the tidiest way to keep them.

Functional Storage MC 1.19.2 · Forge built on Titanium 🗄 one item, one slot

00 — Orientation

Start Here

Functional Storage solves one problem extremely well: storing huge quantities of a single item type per slot, with the count shown right on the block face so a glance tells you how much you have. Build a wall of drawers, frame them up for capacity, link them with a controller, and read or pull from the whole thing through a digital terminal — no channels, no power required for the storage itself.

core

Drawers

1, 2, and 4-slot blocks. Each slot is one item type with a live count. Frame material sets the capacity tier.

automation

Compacting Drawer

Auto-converts nugget ⇄ ingot ⇄ block of one material across its slots — the must-have for ore output.

network

Storage Controller

Links many drawers into one network so insertion and extraction find the right slot automatically.

access

Storage Terminal

A searchable, AE-lite window over every linked drawer — withdraw, deposit, and (with the crafting variant) craft.

Where this fits in the pack

Functional Storage lives in the Storage & Logistics hub alongside the venerable Storage Drawers — they cover the same idea, and you generally pick one. Functional Storage is the more modern, network-and-terminal-forward option; Storage Drawers is the long-standing alternative with its own controller and trim ecosystem. See §07 for a head-to-head.

The shape of a build

  1. Place a drawer. Start with a cheap wood-framed 1×1. Put an item in by right-clicking with a stack; it locks the slot to that type.
  2. Read the count. The face shows the item and its total. Right-click adds one stack; punch (left-click) pulls a stack back out.
  3. Raise capacity. Reframe to iron / gold / diamond / netherite and drop in Storage Upgrades for bigger and bigger per-slot caps.
  4. Build the network. Place a Storage Controller and grow drawers within its reach; everything becomes one inventory.
  5. Add a terminal. A Storage Terminal on the network gives you searchable, AE-style access to all of it.
  6. Automate. Puller/Pusher upgrades, hoppers, or pipes feed the drawers; a Compacting Drawer on each furnace/smeltery output keeps ingots/blocks tidy.

01 — The blocks

Drawers

A drawer is a block divided into slots, each holding one item type and showing its count. Functional Storage ships three slot layouts and a set of framing materials that multiply the base capacity. You choose layout by how many distinct items the spot needs and material by how much of each you want to hold.

Slot layouts

Drawer variants
DrawerSlotsBest for
1×1 Drawer1A single high-volume item — full per-slot capacity in one block.
1×2 Drawer2Two related items side by side (e.g. an ingot and its dust).
2×2 Drawer4Item-variety density — four types in one block, each with its own count.
Capacity splits across slots

A drawer's total volume is shared, so a 1×1 holds the most per item and a 2×2 holds less per slot but four types. Pick the layout for the job: a 1×1 for cobblestone you mine endlessly, a 2×2 for a tidy row of dyes.

Framing materials = capacity tiers

The frame material sets the base capacity multiplier. The numbers below are the familiar defaults (measured in stacks per slot for a 1×1) and are config-adjustable — your pack may differ, so trust the tooltip.

Framing tiers — approximate per-slot capacity (1×1, before upgrades)
FrameTierApprox. capacity
WoodBase≈ 1×
IronII≈ 8×
GoldIII≈ 16×
DiamondIV≈ 32×
NetheriteV≈ 64×

Frames are applied at crafting time, and on most versions you can reframe a drawer in place by combining it with the next material — capacity grows without unpacking the contents.

Using a drawer

  • Insert: right-click an empty slot with a stack to assign that item type, then right-click to add more. Right-click with a full hand dumps your matching stacks in.
  • Extract: left-click (punch) pulls a stack; right-click with an empty hand on most versions pulls one. Sneak-interactions vary per setting.
  • The count display: the face renders the item icon and a live number. A wall of drawers becomes a readable at-a-glance inventory.
  • Locking a slot: use the configuration tool or a key (see §06) to lock a slot to its item so it stays assigned even when emptied — essential for automation that should never re-assign an empty slot to the wrong thing.
Empty but reserved

Locking is what keeps a Puller/Pusher pipeline sane: a locked empty slot still "remembers" cobblestone, so an importer keeps routing cobble there instead of grabbing whatever lands in it first.

02 — Auto-convert

Compacting Drawer

The Compacting Drawer is the standout block. Give it one material and it automatically keeps nugget ⇄ ingot ⇄ block (or seed/crop, or any 9:1 / 4:1 compaction recipe) in sync across its slots, converting on the fly. Insert ingots and it can store the equivalent as blocks; pull nuggets and it breaks a block down. You never manually craft 9-into-1 again.

How it works

  1. Seed it. Put the base item (say, an iron ingot) into the compacting drawer to set its material family.
  2. It maps the tiers. The drawer exposes the related forms it knows — nugget, ingot, block — each as its own readable slot.
  3. Insert at any tier. Drop in blocks, ingots, or nuggets; the drawer normalises everything into one shared pool and shows each tier's count.
  4. Extract at any tier. Pull a block and it consumes 9 ingots' worth; pull nuggets and it cracks an ingot. The math is handled for you.
The must-have for ore output

Park a Compacting Drawer on every furnace, smeltery, or smelting-factory output. Ingots flow in, and you can always withdraw whichever form you need — blocks for storage and building, ingots for crafting, nuggets for fiddly recipes — without a single crafting-grid step.

One family per drawer

A compacting drawer holds one material's tiers, not arbitrary items. Use one per metal. Recipes that compact (3×3 and 2×2) are read from the game's data, so anything with a valid block/ingot/nugget relationship works — including modded metals and many crops.

03 — Boosters

Upgrades

Drawers have upgrade slots (open the drawer's GUI to access them). Upgrades raise capacity or add behaviour. Capacity upgrades stack within the drawer's limit; behaviour upgrades each do one job.

Storage (capacity) upgrades

These multiply the per-slot capacity on top of the frame tier. They come in their own ladder and stack — slot several in and the multipliers compound up to the drawer's cap.

Storage upgrade tiers — approximate per-upgrade multiplier
UpgradeApprox. multiplier
Copper Upgrade≈ 2×
Gold Upgrade≈ 4×
Diamond Upgrade≈ 8×
Netherite Upgrade≈ 16×

Behaviour upgrades

Function upgrades
UpgradeEffect
Puller UpgradePulls matching items out of an adjacent inventory into the drawer/network.
Pusher UpgradePushes items out of the drawer into an adjacent inventory (e.g. feed a machine).
Collector UpgradeVacuums up matching item entities dropped near the drawer.
Void UpgradeSilently voids overflow once the slot is full — perfect for endless cobble/dirt.
Redstone UpgradeEmits a redstone signal based on fill level (high/low/comparator-style control).
Creative UpgradeMakes a slot an infinite source/sink of its item (creative/testing).
How upgrades stack

Capacity upgrades multiply together and combine with the frame tier, so a netherite-framed drawer packed with diamond/netherite storage upgrades holds an enormous amount per slot. Behaviour upgrades each occupy a slot and apply independently — you can mix, say, a Puller and a Void upgrade in the same drawer to "import and cap" a resource.

Mind the slot count

Each drawer has a finite number of upgrade slots. If you want both maximum capacity and several behaviours, you may run out of room — that's a nudge toward putting bulk capacity on dedicated drawers and behaviour on others, all unified by the network.

04 — Linking it all

The Network

A single drawer is fine; a network is the point. The Storage Controller binds every drawer in its reach into one virtual inventory, so a pipe, hopper, or terminal talks to the controller and the network finds the right slot to deposit into or extract from.

Storage Controller

Place a Storage Controller and it claims drawers within a cubic reach around it (commonly a range on the order of ~12 blocks each direction, config-adjustable). Connected drawers turn into one inventory: insert an item anywhere on the network and it lands in the slot already holding that type.

Controller Extension

When a single controller can't see every drawer, add Controller Extensions. Placed adjacent to the controller (or to another extension), they relay the network's reach further out, letting one logical network span a much larger drawer wall or several walls.

Network blocks
BlockRole
Storage ControllerThe brain. Unifies all drawers in reach into one inventory and exposes them to pipes/terminals.
Controller ExtensionExtends the controller's reach to cover more drawers; chain them to grow the network.
Drawers (any frame/layout)The actual storage; auto-join when within reach of the controller or an extension.

Auto-feeding the network

Because the controller presents the whole wall as one inventory, automation is simple: point a hopper, item pipe, or a Pusher/Puller upgrade at the controller (or any drawer) and items route to the matching slot. Lock your slots (§01) first so empty drawers don't get hijacked by the wrong item.

No power needed

Unlike a full AE2/RS network, Functional Storage drawers and the controller don't consume energy to store items. The network is purely logical — which makes it cheap to scale and immune to brownouts.

05 — AE-lite access

Digital Access

A drawer wall is readable, but scrolling a hundred drawers to find one item is slow. The Storage Terminal gives you an AE-style window over the entire network: a searchable grid of everything stored, with one-click withdraw and deposit.

Terminal blocks
Block / itemRole
Storage TerminalSearch, scroll, withdraw, and deposit across every drawer on the connected network.
Crafting TerminalThe Storage Terminal plus a built-in crafting grid — pull ingredients and craft without leaving the screen.

Using the terminal

  • Search: type to filter by name; supports the usual modifiers (mod name, tags) familiar from AE/RS terminals.
  • Withdraw: click an item to pull a stack; shift/ctrl-click variants grab more or send straight to your inventory.
  • Deposit: shift-click items from your inventory and the network files each into its matching drawer slot.
  • Craft (Crafting Terminal): set a recipe in the grid and pull components directly from the drawers, then craft and the result can go back to the network.
Portable / remote access

If your version or pack includes it, a portable/armor terminal lets you browse the same network from your inventory anywhere. Where it isn't present, a wall-mounted terminal at your crafting station is the standard setup — pair it with the Crafting Terminal for a true AE-lite bench.

It reflects the drawers

The terminal isn't separate storage — it's a view of the drawers. Everything you see is sitting in a physical drawer somewhere on the network, which keeps things transparent and lets you mix terminal use with direct drawer interaction.

06 — In the hand

Tools

A few held items make drawers far nicer to live with — configuring slots, locking item assignments, and quick-stacking your inventory into the network in one click.

Functional Storage tools
ToolDoes
Configuration / Linking ToolRight-click drawers to configure slots, link/inspect network membership, and toggle behaviour.
Drawer KeyLocks (and unlocks) a slot to its current item so it stays assigned even when empty — the automation safeguard.
Quick-stack (to network)Empties matching items from your inventory straight into the drawers/network in one action.

Workflow

  • Lock before you automate. Run the Drawer Key across a freshly assigned wall so a Puller can never re-home an empty slot to the wrong item.
  • Use the tool to inspect. The configuration/linking tool tells you what a drawer is joined to and lets you tune per-slot settings without opening menus repeatedly.
  • Quick-stack on return. After a mining trip, one quick-stack action dumps everything the network already knows into its slots, leaving only the new stuff in your bag.
Lock first, regret never

The single most common drawer-automation bug is an empty unlocked slot grabbing the wrong item. Locking with the Drawer Key fixes it permanently — make it a habit.

07 — Living with it

Tips & Comparison

Functional Storage rewards a little discipline. A handful of habits keep a drawer base fast, readable, and stall-free — and it helps to know exactly where it sits relative to the classic Storage Drawers mod.

  • One item per slot
  • Lock before automating
  • Compacting on smelt output
  • Terminal for search
  • Void for endless junk
  • No power to store

Habits that pay off

  • One item per slot, on purpose. The whole value is readability — resist cramming. A 2×2 with four well-chosen types beats a messy chest every time.
  • Compacting drawers on outputs. Put a Compacting Drawer on every furnace/smeltery/smelting-factory output so metals self-organise into nugget/ingot/block and you can always pull the form you need.
  • Pair the terminal for searchable access. Once you pass a couple dozen drawers, a Storage (or Crafting) Terminal turns the wall into an AE-lite inventory you actually want to use.
  • Void the floods. Cobble, dirt, gravel, and netherrack belong in a void-upgraded drawer so they cap out and stop clogging your pipes.
  • Reframe, don't rebuild. Upgrade capacity in place by reframing and stacking storage upgrades rather than tearing the wall down.

Which to pick

They overlap heavily — most players run one or the other. Choose Functional Storage for the modern feel and the built-in terminal; choose Storage Drawers if your pack leans on its established trim/template ecosystem or add-ons. Either way, the core ideas in this guide — one item per slot, framing for capacity, a controller for the network — carry across. Back to the Storage & Logistics hub for the bigger picture.
