A working field manual for Pipez — the no-fuss logistics mod that moves Items, Fluids, Energy and Gas through tidy, tiered pipes with clean per-side configuration. No channels, no cabling rituals, no spaghetti: just point a pipe at a chest or machine, set one face to pull, and let it run. 00 — Orientation

Start Here

Pipez is the modern, low-lag alternative to complicated logistics networks. It moves Items, Fluids, Energy (Forge Energy / FE) and Gas (Mekanism chemicals) through simple pipes you can configure face-by-face. There is no network to register, no channel budget, and no controller block — you place a pipe, open a connection with the wrench, and tell that single face what to do.

items

Item Pipe

Pull from chests & machine outputs, push into inventories, with filters and distribution modes.

fluids

Fluid Pipe

Move water, lava, oil and any mod fluid in millibuckets between tanks and machines.

energy

Energy Pipe

Carry Forge Energy (FE) between any generators, machines and batteries.

gas

Gas / Universal Pipe

Carry Mekanism gas/chemicals; the Universal pipe can do all four resource types in one block.

The whole idea in one line

One pipe family per resource, five tiers per family, and a small configure GUI on every connection. Master that GUI — Disconnect / Pull / Push, a distribution mode, a priority, redstone control, and filters — and you have learned the entire mod.

Where this fits in your base

Pipez is the glue between hubs. See the Storage hub for chests, drawers and digital systems; the Logistics hub for routing and sorting concepts; and the Automation hub for machine setups. Pipez is what physically carries items, fluids, power and gas between all of them without you needing a full storage network.

The shape of the climb

  1. Craft a Basic pipe of the type you need (Item, Fluid, Energy, or Gas) and run a line from a source to a destination.
  2. Wrench the source face to open its configure GUI and set it to Pull (extract).
  3. Pick a distribution mode (Round Robin to balance, Nearest for a single target) so items land where you want.
  4. Add a filter if you only want certain items/fluids to move, and a redstone rule if you want a switch.
  5. Upgrade the tier (Improved → Advanced → Ultimate → Infinity) when one pipe needs to carry more per tick or hold more filter slots.
  6. Reach for XNet/AE2 only when you genuinely need logic or searchable storage — for plain "move this there," Pipez wins on simplicity.

01 — One pipe per resource

Pipe Types

Pipez gives you a dedicated pipe family for each resource, plus a Universal pipe that handles everything. A pipe only carries its own resource — an Item Pipe will not move fluids — so you run parallel lines (or use Universal) when a machine needs more than one input.

The Pipez families
PipeCarriesConnects to
Item PipeItems (stacks)Chests, barrels, drawers, machine item slots, hoppers — anything with an inventory.
Fluid PipeFluids (mB)Tanks, fluid machines, cauldrons, and any block exposing a fluid handler.
Energy PipeForge Energy (FE)Generators, machines, energy cubes/batteries — any FE storage or sink.
Gas PipeMekanism gas / chemicals (mB)Mekanism chemical tanks, machine gas ports, and other chemical handlers.
Universal PipeItems, Fluids, Energy & Gas togetherEverything above — one pipe block that moves all four resource types at once.

Connections form automatically when a pipe touches a compatible block, and each connection face can be set independently (see §03). A pipe with nothing useful on a face simply leaves it inert — no power draw, no lag.

When to use Universal

The Universal Pipe is handy where space is tight or a machine needs several resources from the same direction (e.g. items in, power in, gas in). For long trunk runs it can be simpler to keep dedicated pipes so each line's filters and modes stay obvious at a glance.

02 — Throughput ladder

Tiers

Every pipe family comes in five tiers. Climbing the ladder raises the transfer rate per operation and the number of filter slots available on a connection. Craft a higher tier directly, or upgrade by replacing the pipe and re-wrenching the face.

Tier ladder — rates are approximate, confirm in the GUI / JEI
TierRelative throughputFilter slots
Basiclowestfew
Improved↑more
Advanced↑↑more
Ultimate↑↑↑many
Infinityeffectively unlimitedmost

Each tier moves roughly an order more than the one below it — concretely, an Item Pipe carries more items per operation, a Fluid/Gas Pipe more mB per tick, and an Energy Pipe more FE per tick. The exact numbers are configurable per pack, so treat the table as relative and read the connection GUI for the real figure.

Numbers are approximate

Transfer rates and filter-slot counts here are approximate and pack-configurable. The Infinity tier is the top rung — built for "I never want to think about throughput again" setups like Mekanism 5× plants or Create contraptions. When precision matters, the in-game tooltip and JEI are ground truth.

  • Higher tier = more per tick
  • Higher tier = more filter slots
  • Same GUI on every tier
  • Upgrade by re-placing the pipe

03 — The connection GUI

Per-Side Configuration

The heart of Pipez is the small configure GUI you open by wrenching a connection. Everything a pipe does is decided per-face: whether the face is disconnected, pulling, or pushing; how it spreads the load; which destination wins ties; and whether redstone gates it.

Face state

Three states for every connection
StateWhat it does
DisconnectedThe pipe ignores this side entirely — no transfer, no connection rendered.
Pull (extract)The pipe extracts from the block on this face and carries it into the network. This is your source face.
PushThe pipe inserts into the block on this face. Most destination faces can stay on default insert; set Push explicitly when you want to force output here.

Distribution mode

When a pulled resource could go to several destinations, the distribution mode decides where it lands:

How the pipe spreads its load
ModeBehaviour
Round RobinCycles through valid destinations evenly — the balancer. Best for feeding many identical machines.
NearestFills the closest valid destination first. Predictable; good for a primary target with overflow behind it.
FurthestFills the most distant destination first — the inverse of Nearest.
RandomPicks a valid destination at random; rough load-spreading without strict ordering.

Priority & redstone

Each connection also carries a priority value: when modes tie, higher-priority destinations are served first, so you can force a main machine to fill before an overflow chest. And every face has a redstone control setting — ignore redstone, run only when powered, or run only when unpowered — letting you gate any extract or insert with a lever, comparator, or logic signal.

Set the source, not the whole pipe

You don't configure "the pipe" — you configure each connection. The usual pattern is: leave the long middle of the run untouched, set the one face touching your source to Pull, and let destinations accept the default insert. Less to think about, fewer mistakes.

04 — Choosing what moves

Filters & Upgrades

Filters let a connection move only the resources you choose. The number of filter slots scales with the pipe tier (§02), so an Ultimate or Infinity pipe can run a long, specific list where a Basic pipe handles just a couple of entries.

Filtering controls
ControlWhat it does
WhitelistOnly the listed items/fluids/gases are allowed to move on this connection.
BlacklistEverything except the listed entries is allowed.
Match by itemFilter on the exact item/fluid/gas.
Match by tagFilter by a tag (e.g. all ingots, all logs) so families move together.
Match by NBTDistinguish items by their NBT data (enchanted vs. plain, charge level, etc.).
FuzzyLoosen matching so close variants count as the same entry — handy for damaged/enchanted items.
Filters need redstone (the upgrade)

In Pipez, the precise filter list is provided by a filter that costs redstone to make — the per-connection filter is the "upgrade" you slot to gain item/tag/NBT control beyond the simple defaults. Build a small stock of redstone before you plan a heavily-filtered network, and remember filters apply per connection, so each face can sort differently.

Per-channel filtering

Because filtering lives on the connection rather than on a global network, a single trunk line can sort many ways at once: one destination face whitelists ingots, another whitelists dusts, a third takes the blacklisted remainder. There is no shared channel to clash — each face is its own little router.

Tag & fuzzy save slots

Prefer tag matching over listing items one by one — "all ingots" is one entry instead of a dozen. Turn on fuzzy when NBT differences (durability, enchantments) would otherwise split one logical item into many filter entries.

05 — Building lines

Wrench & Routing

The wrench is how you talk to Pipez. Use it to open a connection's configure GUI, to set which side extracts, and to toggle or rotate connections. A clean base is mostly the discipline of setting one extract side per source and letting a single line feed everything downstream.

The wrench workflow

  1. Run the line. Place pipes from your source to your destination(s). Connections to inventories form automatically.
  2. Open the source face. Wrench the connection touching your chest or machine output to bring up the configure GUI.
  3. Set it to Pull. Mark that single face as the extract side. This is the only "active" decision most lines need.
  4. Tune distribution. Choose Round Robin to balance across machines, or Nearest to favour a primary target.
  5. Filter & gate if needed. Add a filter on the destination faces and a redstone rule where you want a switch.

Chaining machines on one line

A single Pipez line can feed many machines in series. Run the pipe past each machine, let each tap accept the default insert (with its own filter if the machines want different inputs), and use Round Robin from the source so the line spreads work evenly. Because each connection filters independently, the same trunk can deliver ore to one machine and fuel to another without crossing streams.

One extract side per source

The golden rule: each source block should have exactly one Pull face. Multiple extract faces on the same inventory fight each other and make behaviour hard to read. Extract once, then let routing and filters do the rest down the line.

06 — Universal glue

Cross-mod

Pipez speaks the common interfaces — Forge Energy, fluid handlers, item handlers, and Mekanism's chemical system — so it moves resources between machines from any mods that expose them. That makes it ideal connective tissue in a mixed modpack.

What Pipez bridges
ResourcePipeTypical cross-mod use
Mekanism gas / chemicalsGas PipeCarry oxygen, hydrogen, sulfuric acid, etc. between Mekanism machines without Pressurized Tubes.
Forge Energy (FE)Energy PipePower any FE machine from any FE generator — Mekanism, Thermal, Powah, Create-adjacent and more.
FluidsFluid PipeMove water, lava, oils and mod fluids between tanks and machines across mod boundaries.
ItemsItem PipeShuttle items between any inventories — vanilla chests, drawers, machine slots, storage blocks.

Because it relies on standard interfaces rather than mod-specific networks, Pipez often "just works" where a native pipe from one mod refuses to connect to a machine from another. Use it as the universal glue between mod machines: Mekanism gas here, Thermal FE there, Create fluids in between.

Gas needs Mekanism

The Gas pipe carries Mekanism's chemical system specifically, so Gas/chemical transfer assumes Mekanism (or a mod that uses its chemical API) is installed. FE, fluids and items rely on the broadly-implemented Forge capabilities and work with almost anything.

07 — Field notes

Tips

A handful of habits that keep Pipez networks fast, readable, and lag-free.

  • Set ONE extract side per source. Multiple Pull faces on the same inventory compete and confuse routing. Extract once, route downstream.
  • Use Round Robin to balance. When feeding several identical machines, Round Robin spreads the load evenly so nothing starves while another backs up.
  • Bump to Ultimate / Infinity for high throughput. Mekanism 5× plants and Create contraptions move a lot per tick — a Basic pipe will bottleneck them. Higher tiers carry far more and give you more filter slots too.
  • Filter by tag, not by item. One "all ingots" tag entry beats a dozen individual items and survives new additions to your pack.
  • Gate with redstone where you want a manual or automatic switch — keep-stock control, night-only pumping, comparator thresholds.
  • Prefer XNet / AE2 only when you need logic or storage. If the job is "move this there," Pipez is simpler and lighter. Reach for a full storage/logic network when you need searchable inventories, autocrafting, or conditional routing Pipez can't express.
The honest scope

Pipez is deliberately small. It will not auto-craft, search a network, or run complex conditional logic — and that's the point. For point-to-point transfer of items, fluids, power and gas across any mods, it is about the lowest-friction, lowest-lag option you can install. Use it for the plumbing; bring in the heavyweight networks only where they earn their keep.

Don't over-build

A common trap is wrapping every machine in Infinity pipes and complex filters when a Basic Item Pipe with one Pull face would do. Start simple, upgrade the specific lines that actually bottleneck, and you'll keep both your sanity and your tick rate.
