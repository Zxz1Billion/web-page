The guardrails (please read first)

Everything on this page was filtered against four rules so it's an easy yes/no for you.

1 · No world reset

Every mod here is safe to add to the existing save. Some mods add new structures, mobs, ores, or wild crops — where they do, that content only appears in newly-generated chunks (areas players haven't explored yet). That needs no regeneration; it just means already-explored terrain stays as-is. Each such mod is marked new chunks. Anything that would genuinely require a fresh world was excluded (see §Compatibility).

2 · 1.19.2 Forge only · 3 · No tech-spine overlap · 4 · Performance-aware

Each pick is confirmed for 1.19.2 Forge. Anything that overlaps the pack's existing AE2 / IC2 systems, or adds a parallel power/storage system, is flagged so you can rule on it deliberately. And because the pack runs Distant Horizons + Rubidium, the proposal leans into performance helpers and flags the few heavier picks.

Compatibility, performance & what was excluded

The decisions worth your attention before flipping switches.

Excluded for needing fresh terrain

Tinkers' Reforged was left off: it adds new ores (Yttrium/Barium/Thallium across the dimensions), which only generate in new chunks — so its materials would be effectively unreachable in your explored world without regenerating. It only makes sense on a fresh world, which breaks the no-reset rule. Applied Flux was also dropped — no confirmed 1.19.2 Forge build.

Refined Storage vs the pack's AE2

Both are endgame digital-storage networks covering the same niche. They coexist without crashing, but running both splits progression. Recommendation: keep AE2 as the storage endgame and treat RS as a deliberate choice — not a default add.

Don't stack parallel systems

The pack commits to AE2 + IC2. Mods that add new parallel systems — Create (rotation power), Tom's Simple Storage (mini-ME) — are technically fine but dilute focus. Treat each as a deliberate "yes, we want this pillar."

Structure spacing (new-chunk mods)

If you add When Dungeons Arise and/or Dungeons and Taverns on top of the YUNG suite, tune their spacing/separation configs so structures don't overlap-generate in fresh chunks (avoids ugly collisions and world-gen lag). None of this touches existing terrain.

Distant Horizons performance

Heaviest risk is Create (contraptions/block entities). The §07 performance mods (spark, FerriteCore, ModernFix, Entity Culling, Clumps) actively offset the cost of everything else and are the lowest-risk additions on this page.

Almost Unified scope

The pack's Almost Unified unifies tagged ores/ingots/gems — not decoration or machine recipes. Manage block/recipe overlap (Quark stones, Chipped variants) via each mod's config, not unification. Remember to disable Quark's ore world-gen.

Before installing

Confirm each mod's exact 1.19.2 Forge file on CurseForge and pull required libraries (Placebo, Moonlight, Balm, Citadel, Athena, Architectury, FTB Library, Sophisticated Core, AutoRegLib) — CurseForge resolves most automatically. A test world is wise before touching the live save, but none of these picks require a reset.
