The One-Block
Router

Modular Routers does the work of a tangle of hoppers, pipes, and droppers inside a single block. You drop Modules into the router's grid, point it at an inventory or a spot in the world, and it pulls, pushes, drops, vacuums, breaks, places, and clicks — all on one tidy tick loop. This guide runs from your first Item Router to a power-fed auto-cobble farm and a self-building bridge.

00 — Orientation

Start Here

Modular Routers replaces piles of vanilla logistics with one programmable block. The Item Router holds a small inventory (its buffer) and a grid of Modules; each tick it runs its modules in order, and each module performs one action — pull from a chest, push to a furnace, drop items, break a block, suck up the floor. Configure the modules, feed it items (and optionally power), and a single block automates what used to need a wall of hoppers.

block

Item Router

The brain. Holds a buffer inventory plus a grid of module slots and an upgrade slot. One block, placed against whatever you want it to act on.

behaviour

Modules

Each module is one verb — Puller, Sender, Dropper, Breaker, Placer, Vacuum, and more. Slot several into one router to chain actions.

control

Upgrades

Speed, Stack, Filter, Camouflage, Blast, Security, and friends tune the router's throughput, targeting, and looks.

tuning

Tick & Eco

The router ticks on a configurable interval and supports redstone control and an eco/idle mode so it isn't churning when there's nothing to do.

Where this fits in your base

Think of Modular Routers as the precise, compact end of Storage & Logistics. For single, well-defined jobs — restock this machine, vacuum this drop zone, mine this column — one router beats a pipe network. For moving bulk across a whole base, you'll still reach for a dedicated transport mod (see §07). The two coexist happily.

The shape of a router

  1. Place the Item Router. It has a front face (the working/target side) — placement direction matters for some modules.
  2. Open it and add Modules. Drag modules into the grid; each will act every tick in slot order.
  3. Configure each module. Set its filter, target, and direction inside the module's own screen.
  4. Feed it. Pipe or hopper items into the buffer; add power if you use energy-hungry upgrades.
  5. Tune the tick. Add a Speed Upgrade for throughput, or use redstone / eco mode to keep it idle until needed.

01 — The block itself

The Router

The Item Router is the only block you ever place. Everything else lives inside it. It exposes three things: a buffer (a small internal inventory items pass through), a grid of module slots, and an upgrade slot. Open the GUI and you'll see the buffer on one side and the module/upgrade grid on the other.

Buffer & module slots

Items the router handles flow through the buffer — a Puller deposits pulled items here, a Sender pushes items out of here. Modules execute top-to-bottom, left-to-right each tick, so order can matter (pull before you send, break before you vacuum). The number of usable module slots is fixed by the block, and each module carries its own filter and target settings independently.

Tick rate & eco mode

The router runs on a tick interval — by default a modest number of ticks between actions, so it's gentle on the server. Two controls keep it efficient:

  • Redstone control — set the router to run always, only when powered, only when unpowered, or on a pulse (single action per rising edge). Pulse mode is perfect for "do exactly one thing when I flick the lever."
  • Eco / idle mode — when the router has had nothing to do for a while it backs off to a slower idle tick, then snaps back to full speed when work appears. This cuts needless tick churn on routers that only occasionally have items.
Slot order is a tool, not a gotcha

Because modules fire in grid order, you can sequence behaviour in one router: a Breaker in slot 1, a Vacuum in slot 2, and a Sender in slot 3 will mine a block, hoover the drops, and ship them onward in the same tick window. Rearrange modules to change the pipeline.

Upgrades at a glance

Upgrades go in the router's upgrade slot (most stack up to a cap). They change how the whole router behaves, not what a single module does. The big ones:

Router upgrades — what each one changes
UpgradeEffect
Speed UpgradeFaster tick rate — more actions per second. Stacks; costs energy if combined with Blast.
Stack UpgradeMove a whole stack per action instead of one item. The single biggest throughput boost.
Filter UpgradeLets a module filter against the contents of a real inventory/tag set as a reference — "match whatever is in that chest."
Camouflage UpgradeDisguise the router as any full block so it blends into a wall or floor.
Blast UpgradePushes the router past the normal speed cap by spending energy (FE) per operation.
Security UpgradeLocks the router to its owner so others can't open or tamper with it.
Fluid UpgradeEnables the fluid modules to move liquids in/out of the router's tank.
Sync UpgradeSynchronises the router's actions to a redstone signal / clock for precise timing.
Muffler UpgradeSilences sounds from modules (e.g. the Breaker's mining noise).

Upgrades are covered in depth in §05; their energy implications in §06.

02 — The verbs

Modules Overview

Modules are the heart of the mod — each is a small item you craft and slot into a router. A module is one action, configured with its own filter and target. Combine several in one router to build a pipeline. Here's the core roster.

Movement modules

Pulling, sending, and dropping items
ModuleWhat it does
PullerPulls items into the router's buffer from the inventory it faces (chest, machine output, etc.).
Sender 1Pushes items out to the adjacent inventory on the router's facing side — short range, the cheapest sender.
Sender 2Pushes to a target within a limited range along the facing direction (no direct contact needed).
Sender 3Pushes to a target set by coordinates, across a longer range (and, in many configs, across larger distances).
DistributorSpreads items across multiple destinations (round-robin / split / fill modes) instead of one target.
DropperEjects items into the world as dropped entities on the facing side (like a vanilla dropper).
FlingerLaunches items as projectiles, throwing them across a gap or into a collection point.

World-interaction modules

Acting on blocks and the environment
ModuleWhat it does
BreakerBreaks the block in front of the router (respecting tool/silk-touch settings) and drops the result into the buffer.
PlacerPlaces a block from the buffer into the world space in front of the router.
ExtruderPlaces blocks in a line to build outward (a bridge/wall), and can retract them — great for auto-builders.
VacuumSucks up nearby dropped item entities into the buffer over an area in front of the router.
ActivatorPerforms a simulated right- or left-click — use items, till soil, shear, harvest, attack — on the facing block/space.
PlayerMoves items to/from the owner's player or ender inventory (remote access to your own pockets).
DetectorEmits a redstone signal based on the router's buffer contents (count thresholds) — logic and auto-stock control.

Fluid modules

With the Fluid Upgrade installed, the router gains a small tank and the fluid modules become useful:

  • Fluid 1 / Fluid 2 — pull liquids into the router's tank, or push them out to an adjacent or ranged fluid handler, mirroring the item Puller/Sender split.
Mix and match in one block

A single router can hold a Puller and a Sender and a Detector at once. That's the whole point: one router can pull from a machine's output, filter the result, ship the good stuff onward, and emit redstone when a buffer fills — without a single pipe leaving the block.

03 — Aiming the router

Filters & Targeting

Two settings turn a generic module into a precise tool: the filter (which items it acts on) and the target (where it acts). Both are configured per-module, so two modules in the same router can filter and aim completely differently.

Filters

Every movement and world module carries a filter list you can run as a whitelist (only these) or blacklist (everything but these). You can populate it several ways:

Ways to define what a module matches
Filter typeMatches
By itemSpecific item stacks dropped into the filter slots.
By tagAn item tag (e.g. forge:ores, forge:ingots) so a whole category matches at once.
By modEverything from a given mod id — handy for "all the Create stuff."
Regex / nameA regular-expression match against item names for flexible patterns.
InspectionMatch by NBT/durability/inspection criteria for damaged tools, enchanted items, etc.
Mod filter via referenceWith the Filter Upgrade, mirror the contents of a reference inventory as the live filter.
Whitelist vs blacklist

Flip the toggle per module. A Puller with a blacklist of your fuel item will pull everything except the fuel out of a furnace (leaving it lit); a Sender with a whitelist of one ingot keeps a sorting line clean. Tag and mod filters keep the list short as your item count grows.

Targeting

How a module chooses where to act depends on the module:

  • Look-at / facing — most world modules (Breaker, Placer, Activator) act on the block directly in front of the router's face.
  • Area — the Vacuum covers a small region in front of the router; you set how it sweeps.
  • Adjacent inventory — Pullers and Sender 1 act on whatever inventory touches the relevant face.
  • Ranged / coordinate — higher Senders aim further afield, which leads to the key distinction below.

How the Sender ranges differ

Sender 1 / 2 / 3 — range and how you target
SenderRangeHow you aim it
Sender 1Touching onlyPushes into the inventory directly on the facing side — zero configuration.
Sender 2Short, line-of-sightFires along the facing direction up to a limited block range; finds the first valid inventory.
Sender 3Long / coordinateYou set explicit target coordinates (often via a Sync/marker tool); reaches across long distances.

As a rule: use Sender 1 when the chest is right there, Sender 2 to reach across a small gap without a pipe, and Sender 3 when the destination is somewhere specific and far away.

04 — Putting it to work

Common Builds

A handful of recipes cover most of what people install Modular Routers for. Each is one or two routers; the magic is in the module mix.

Auto-cobblestone generator

Build a standard water+lava cobble generator so a fresh block forms in front of the router's face. Then:

  1. Place the Item Router facing the generation spot.
  2. Add a Breaker module — it mines the cobble as it forms and drops it into the buffer.
  3. Add a Sender 1 module facing a chest — it ships the cobble out of the buffer.
  4. Add a Speed Upgrade (and a Stack Upgrade) so it keeps pace with regeneration and moves full stacks.
  5. Optional Muffler Upgrade to silence the constant mining sound.

One router, one chest, infinite cobble — no quarry, no pipes.

Vacuum item collector

For a drop zone (mob farm output, a crop trampler, an item-spitting machine), a Vacuum module sweeps dropped entities into the buffer, and a Sender ships them to storage. Add a Filter so it only grabs the loot you care about and leaves XP orbs / junk alone. This single router replaces a whole floor of hoppers.

Auto-builder bridge (Extruder)

Load the buffer with building blocks, point the router across the gap, and an Extruder module lays a line of blocks outward to form a bridge or wall — and can retract them on a redstone signal. Pair it with redstone control (pulse mode) so each lever flick extends one length.

Mob-farm collector

Under a mob farm, combine a Vacuum (grabs drops) with a Distributor (splits the haul across several storage chests) so no single chest overflows. A Detector module can emit redstone when a particular drop (e.g. gunpowder) crosses a threshold to trigger downstream automation.

Restocking a machine

To keep a furnace, smelter, or modded machine fed: a Puller drains the machine's output slot into the buffer, a Sender pushes fuel/input back in (with a whitelist for the correct slot/item), and a Detector can pause everything via redstone when the output chest is full. The whole restock loop lives in one block hugging the machine.

Filter the Puller, not just the Sender

When restocking, blacklist the machine's fuel on the Puller so you only pull finished product and never accidentally rob the furnace of its coal. Small filter choices like this are what make a one-block automation feel reliable.

05 — Tuning the block

Upgrades Deep-Dive

Upgrades change the router as a whole. A few of them define how it scales, so they're worth understanding rather than just stacking blindly.

Stack & Speed — the throughput economy

These two solve different halves of "go faster":

  • Stack Upgrade raises how much moves per action — from one item toward a full stack. This is usually the first upgrade you want, because moving 64 at once beats ticking 64 times.
  • Speed Upgrade raises how often the router acts by shortening the tick interval. Stack a few for snappy, responsive routers.

For bulk transfer, Stack first, then Speed: a stack-per-tick router at a moderate speed outruns a single-item router spinning frantically, and it's far lighter on the game.

Blast — past the speed cap, for a price

Speed Upgrades alone hit a cap. The Blast Upgrade breaks through it by spending energy (FE) per operation — the router must be connected to a power source. Use it when you genuinely need extreme throughput (a high-rate cobble or mob farm) and have spare FE; skip it on idle utility routers where the energy cost isn't worth it.

Camouflage — hide the router

The Camouflage Upgrade lets you disguise the router as any full block, so it vanishes into a stone wall or a wooden floor. Purely cosmetic, but it's the difference between a clean build and a box of grey cubes poking out of your base.

Sync — line up with redstone

The Sync Upgrade ties the router's actions to a redstone signal or external clock so it fires exactly when you want, in step with other machinery. Combine with pulse redstone control for one-action-per-pulse precision (auto-builders, timed dispensers).

Security — keep others out

On multiplayer servers, the Security Upgrade binds the router to its owner: other players can't open the GUI, pull its modules, or break it without permission. Worth adding to any router holding valuable modules or guarding access to your storage.

Upgrades stack — to a cap

Most upgrades stack (more Speed = faster, more Stack = bigger batches) but each has a per-router maximum, and they share one upgrade area, so you're budgeting slots. Check the in-game tooltips for the exact caps in your pack — they're configurable and JEI/the tooltip is the ground truth.

06 — Power & thrift

Energy & Eco

Most routers run for free — a plain Puller/Sender setup needs no power at all. Energy only enters the picture when you push for speed beyond the normal limits.

When the router wants FE

  • Blast Upgrade spends FE per operation to exceed the speed cap — connect a cable and feed it.
  • Heavy Speed stacks can draw energy in configurations that meter fast ticking, so a strongly-overclocked router benefits from a power feed even where it isn't strictly required.

A router with energy upgrades buffers a little FE internally and draws from any adjacent FE source. If it runs dry, it simply falls back toward its un-boosted rate rather than stopping dead.

The eco / redstone modes — saving power and ticks

Even on free routers, you usually don't want them hammering the tick loop when there's nothing to do. Use:

Throttling a router
ModeUse it when…
Eco / idle modeThe router only occasionally has work — it idles slow and wakes up to full speed on demand.
Redstone: poweredYou want it to run only while a signal is held (gated by a lever, comparator, or sensor).
Redstone: unpoweredDefault-on logistics you can pause with a signal.
Redstone: pulseOne action per rising edge — precise, on-demand single operations.
Balance throughput against cost

The honest tuning order is: Stack Upgrade for batch size, a couple of Speed Upgrades for responsiveness, and only then Blast + FE if you still need more. Pair any of it with eco/redstone gating so the router isn't burning ticks (or energy) when its job is done. You get the throughput where it matters and a quiet base where it doesn't.

07 — Field notes

Tips & When Not To

Modular Routers shines at compact, specific automation. A few habits make it sing — and a couple of honest limits tell you when to reach for something else.

Get the most from one block

  • One router replaces many hoppers. A Puller + Sender pair does what a row of hoppers does, but filtered, faster, and in a single block you can camouflage away.
  • Combine modules in one router. Don't reach for a second block until you've used the slots you have — break, vacuum, filter, and send can all live together and fire in slot order.
  • Filter early. Filtering on the Puller keeps junk out of the buffer entirely, which is cleaner and cheaper than filtering on the way out.
  • Use Detector + redstone for logic. Let a router emit a signal on a threshold to start/stop other machinery instead of bolting on a separate comparator setup.
  • Pulse mode for precision. When you want exactly one action per trigger (an Extruder bridge, a one-shot dispenser), pulse redstone control beats a free-running router.

When to prefer a dedicated transport mod

Routers are point-tools, not a base-wide network. Reach for something else when:

  • You're moving bulk across the whole base. For a sprawling network of many sources and sinks, a pipe mod like Pipez (simple tiered pipes) or XNet (one controller driving many connections) is built for that scale.
  • You need a true logistics graph. XNet's channels and a storage system handle complex routing and crafting far better than chaining Senders.
  • You want fewer block entities. A handful of routers is fine; hundreds of them each ticking is heavier than one XNet controller managing the same connections.
The sweet spot

Use Modular Routers for the edges — pull from this machine, vacuum this farm, mine this column, restock that furnace — and a dedicated transport mod for the trunk. One router at each automation point feeding a shared network is the tidiest of both worlds.

Trust the game, not this page

Module names, slot counts, ranges, and energy costs are configurable and vary by pack and version. Treat the numbers here as defaults for MC 1.19.2 Forge — the in-game tooltips and JEI/EMI recipe data are the ground truth in your install.
