The RFTools
Manual

A field manual for RFTools Base — the common library and foundation under McJty's sprawling RFTools ecosystem. Base is the shared core every RFTools module is built on: the Machine Frame, the Smart Wrench, Machine Infusion with Dimensional Shards, and the modular-machine framework. The famous modules — Power, Storage, Control, Builder, Dimensions, Utility — are separate add-ons that may or may not be in your pack, so this guide stays honest about what's actually guaranteed.

RFTools Base MC 1.19.2 · Forge Powered by FE ("RF") ⚠ Modules are optional

00 — Orientation

Start Here

RFTools is a modular tech suite: a family of mods, each adding a focused toolset (power, storage, automation, world-shaping), all bolted onto one shared base. That base is RFTools Base — a small, almost invisible mod that, on its own, doesn't give you a quarry or a dimension builder. Instead it provides the common library the other modules depend on: the Machine Frame used in most RFTools machine recipes, the Smart Wrench for configuring them, Machine Infusion with Dimensional Shards for upgrading machines, and the shared GUI / energy / redstone framework.

Be honest about what's installed

This page is the guide for RFTools Base specifically. Everything below it under "The Modules" — Power, Storage, Control, Builder, Dimensions — is a separate add-on. RFTools Base being present does not mean those are. Treat their sections as "if your pack has this module" and confirm in JEI before you plan a build around them. See §07 for how to check.

Everything in the ecosystem runs on Forge Energy (FE) — historically called RF (Redstone Flux), which is where the "RF" in RFTools comes from. If your pack has a broader automation & power write-up, this manual is the RFTools spoke of that hub: pair Base's machines with whatever power generation and cabling your pack provides.

The shape of the ecosystem

  1. RFTools Base loads first. It registers the Machine Frame, Smart Wrench, Dimensional Shard infusion, and the modular-machine plumbing. The modules below all require it.
  2. Each module adds a domain. Power adds Powercells; Storage adds the Modular Storage / Storage Scanner; Control adds visual programming; Builder/Dimensions add quarrying and custom worlds.
  3. You mix and match. A pack might ship all of them, a few, or just Base as a dependency for some other mod. Read the mod list.
  4. Common patterns carry over. Once you learn one RFTools GUI — energy bar, redstone mode buttons, Smart Wrench behaviour — you know them all.

01 — What Base actually gives you

Core Components

RFTools Base is small on purpose — it's the part every other module reuses. Here is what it contributes and why each piece matters across the whole ecosystem.

The Machine Frame & Machine Base

The Machine Frame is the iconic RFTools crafting component: a hollow cube of frame edges that sits at the centre of most RFTools machine recipes (across the modules), the way a casing or a "machine chassis" does in other tech mods. The Machine Base is the lower foundation block many machines are built on top of. Together they're the shared "you are looking at an RFTools machine" identity — recognisable frame, consistent footprint.

Frames are everywhere

If a module is installed, expect its machines to want a Machine Frame in the recipe. Batch-craft a stack early — almost nothing in RFTools gets built without one, and the recipe is cheap relative to how often you'll need it.

The Smart Wrench

The Smart Wrench is RFTools' multi-tool for its own blocks. It is the single most useful Base item and you should keep one on your hotbar at all times. Depending on its mode (cycle it with a key/right-click as your pack maps it), it can:

  • Configure — open a block's settings or set side I/O without opening the full GUI.
  • Rotate — turn machines to face the way you want.
  • Move / pick up — relocate certain RFTools blocks (and link blocks like Powercells) without losing their settings or stored contents.
Smart Wrench modes (behaviour is mode-dependent; numbers approximate)
ModeWhat it doesTypical use
Select / ConfigureOpens config / sets a side, or links paired blocksWiring up I/O, pairing networked blocks
RotateCycles the block's facingAiming machine fronts & outputs
Move / PickupRemoves the block keeping its data intactRelocating a configured machine cleanly

Machine Infusion & Dimensional Shards

RFTools machines can be infused for better performance. You feed a machine Dimensional Shards (sometimes seen as Dimensional Cells/Crystals depending on the module set) repeatedly through the Machine Infuser until it reaches full infusion. An infused machine is noticeably more energy-efficient and can run faster — the exact bonus is per-machine and approximate, but the rule of thumb is: a fully infused machine costs less FE per operation than an un-infused one.

Infuse the machines you lean on

Dimensional Shards are a limited resource, so spend them where they pay off: the machines that run constantly (your storage scanner, a hard-working Builder, a busy processor). A machine you toggle once a day isn't worth a shard. See §07.

Dimensional Shards / Cells — where they come from

Dimensional Shards are the ecosystem's premium upgrade currency. In packs with the relevant modules they're mined or crafted (often associated with the Dimensions module's worldgen or with end-tier ore), and they double as the fuel/charge medium for some module machinery. If your pack only has Base plus a subset of modules, the source of shards is pack-specific — check JEI for the "Dimensional Shard" usage tree to see how your install expects you to get them.

02 — Why it feels the way it does

The RFTools Philosophy

RFTools has a recognisable design language. Learn it once and every module feels familiar — that consistency is exactly what Base enables.

Clean, information-dense GUIs

RFTools GUIs are famously functional: a clear energy bar, explicit buttons, and labelled fields rather than hidden menus. You can usually tell at a glance whether a machine has power, what mode it's in, and what it's doing. The trade-off is a slightly utilitarian look — but you rarely have to guess.

Redstone modes on (almost) everything

Most RFTools machines expose a redstone control mode, typically cycling through something like: Ignored (always run), Off when powered, and On when powered. This makes RFTools machines trivial to gate with vanilla redstone or with the Control module — no extra adapter block required.

Common redstone modes (labels vary slightly by machine)
ModeBehaviour
IgnoredRuns regardless of redstone — the default for most machines
Off (when powered)Stops while receiving a redstone signal
On (when powered)Only runs while receiving a redstone signal

Energy-aware machines

RFTools machines are built around FE from the ground up: each has an internal buffer, draws what it needs per operation, and many scale their cost with how hard you push them. This is exactly why Machine Infusion (§01) matters — efficiency is a first-class concern, not an afterthought.

The "module" structure

The big idea: instead of one monolithic mod, RFTools is composable. A pack author can include only the modules they want, and because they all share Base, they interoperate cleanly. The Utility module, for instance, is a grab-bag of quality-of-life blocks (teleporters, screens/info panels, environmental controllers, redstone helpers) that round out whatever else is installed.

Base is the enabler, not the payload

If you installed RFTools expecting quarries or dimensions and only see frames, wrenches, and infusion: you have Base without those modules. That's working as intended — Base is the shared foundation. Add the specific module mod to get its features.

03 — Add-on · RFTools Power

Power & Powercells

If RFTools Power is installed Powercells are RFTools' signature power storage and the reason many players love the ecosystem — but they ship in the separate RFTools Power module. If you don't see Powercells in JEI, your pack didn't include it; skip to §04.

Powercells — storage linked by location, not cables

A Powercell network is a set of Powercell blocks that share one combined FE buffer. The clever part: cells don't have to be adjacent or wired. You build a multi-block bank of cells, and any cell linked to the same network draws from and contributes to the same pool — you can even carry a card to link a remote cell to the network, so power placed here is available there without running a cable across the world.

Powercell block roles (capacities are tiered & pack-configurable — read tooltips)
BlockRole
Powercell (Simple / Normal / Advanced)Adds capacity and I/O to the shared network; higher tiers store & transfer more
Network cardLinks a Powercell to an existing network regardless of distance
Card-bound remote cellA cell elsewhere that taps the same pool — wireless-feeling power sharing
The pitch in one line

Powercells = a big shared FE battery you can extend by placing more cells anywhere and linking them, instead of running and balancing cable. Use the Smart Wrench to relocate cells without dumping their charge.

If you don't have it

Without the Power module you'll need your pack's other batteries/capacitors for bulk FE storage. The rest of RFTools Base works fine on any FE source — Powercells are a luxury, not a requirement.

04 — Add-on · RFTools Storage

Storage & Scanner

If RFTools Storage is installed A searchable storage system that's lighter-weight than AE2 or Refined Storage — but it lives in the separate RFTools Storage module. Confirm in JEI before planning around it.

Modular Storage & the Storage Scanner

The concept is a searchable, indexed storage network without channels or autocrafting overhead:

  • Modular Storage blocks hold items in tiered storage modules (the capacity media — bigger modules, more slots), each browsable through one tidy, searchable GUI.
  • The Storage Scanner sits at the heart of it: point it at nearby inventories (its own modular storage, vanilla chests, other blocks in range) and it gives you one unified, searchable view across all of them, letting you withdraw, deposit, and find items from a single screen.
RFTools Storage pieces (ranges & capacities are pack-configurable)
Block / itemRole
Storage ScannerCentral searchable index across in-range inventories; the "terminal"
Modular Storage blockHolds a Storage Module; the indexed item bank
Storage Module (tiered)The capacity media — higher tiers store more item types
Storage Terminal / RemoteAccess the scanner's view from elsewhere
Why pick it over AE2

No channels, no fluix cabling, no crafting CPUs to manage — the Storage Scanner just indexes what's near it. It's the right call when you want "find my stuff fast" without committing to a full digital-storage mod. For mass autocrafting, a heavier mod still wins.

05 — Add-on · RFTools Control

Control & Programming

If RFTools Control is installed Visual, in-world automation programming — in the separate RFTools Control module. Skip if JEI doesn't list the Processor.

Visual programming with the Processor

RFTools Control lets you build automation logic by laying out program cards rather than writing text. The centrepiece is the Processor, which runs programs that you assemble visually from event/action tiles — "when this happens, do that." It can read and control RFTools machines and inventories, react to redstone, move items, and orchestrate multi-step routines.

Control module pieces
Block / itemRole
ProcessorRuns your visual program; the "CPU" of the setup
Program / Graphics cardsHold the program and drive its GUI capacity
Connector nodesWire the Processor to the machines & inventories it acts on
Where it shines

Control is the glue for "smart base" behaviour: conditional crafting, keeping a buffer stocked, sequencing machines, reacting to power or item thresholds. It's a gentler on-ramp to logic than a full computer mod, since the program is a visual flow rather than code.

06 — Add-ons · Builder & Dimensions

Builder & Dimensions

Two separate modules The world-shaping side of RFTools lives in two distinct add-ons — RFTools Builder and RFTools Dimensions. Either, both, or neither may be in your pack. Confirm in JEI.

The Builder (RFTools Builder)

The Builder is a versatile area-operation block driven by Shape Cards that define a 3D region. Depending on the card it can:

  • Quarry — clear/collect a region (with variants that void worthless blocks or keep only ores).
  • Move — physically shift a structure from one place to another.
  • Copy — duplicate a captured shape elsewhere (resources permitting).
  • Build — place a stored shape into the world.

It's FE-hungry and scales cost with the work done, which is exactly the kind of machine that benefits from Machine Infusion (§01) and a solid Powercell bank (§03).

The Dimension Builder (RFTools Dimensions)

RFTools Dimensions lets you create custom dimensions — you compose a dimension from features/dimlets (terrain type, biomes, materials, special effects), pay an energy/resource cost to construct and maintain it, and travel there. It's a sandbox for designed worlds: a quarry dimension, a flat building space, a resource-rich pocket, etc.

Clearly add-on, often absent & version-sensitive

Both modules are separate downloads, and the Dimensions module in particular has historically lagged or changed substantially between MC versions. Do not assume either is present just because RFTools Base is — check JEI, and trust your in-game recipe browser for exactly how your pack's version works.

07 — Field notes

Tips & Checklist

The short, honest version of everything above.

  1. Keep a Smart Wrench on you. Configure, rotate, and move RFTools machines with it — moving a configured block keeps its settings and contents, which saves enormous rework.
  2. Batch-craft Machine Frames. Nearly every RFTools machine across the modules wants one. Make a stack and stop thinking about it.
  3. Infuse the machines you use heavily. Dimensional Shards are limited — spend them on always-on workhorses (busy Builder, Storage Scanner, a hard-running Processor), not on rarely-used blocks.
  4. Use redstone modes instead of extra blocks. Most machines gate themselves on a redstone signal natively — no adapter needed.
  5. Check JEI first — every time. The single most important habit. It is the ground truth for which RFTools modules are installed and how their recipes work in your pack.
How to confirm which modules you have

Open JEI and search for a signature block per module: Powercell (Power), Storage Scanner (Storage), Processor (Control), Builder / Shape Card (Builder), Dimension Builder / Dimlet (Dimensions). No result means that module isn't installed — and any section above marked "if installed" simply doesn't apply to you.

Don't plan around features you can't see

RFTools Base alone gives you frames, the Smart Wrench, and Machine Infusion — not quarries, dimensions, Powercells, or storage scanners. Those are the add-on modules. If a build plan depends on one of them, verify the module is present before you commit resources. When this guide and your install disagree, the in-game JEI/tooltips win.
