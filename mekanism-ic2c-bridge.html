The EU/FE
Bridge

A drop-in patch that lets Mekanism (Joules / Forge Energy) and IC2 Classic (EU) trade power cleanly on the To the Moon 2.0 server. The two mods speak different units on different networks; the common language is Forge Energy, and the whole job is locking one consistent exchange rate so no power is created or destroyed when it crosses the line.

1 EU = 4 FE = 10 J Mekanism 10.3.9.13 IC2 Classic 2.1.3.3 Config-only · no mixin

00 — Orientation

The bridge, in one line

Mekanism runs on Joules and speaks Forge Energy (FE/RF) natively. IC2 Classic runs on EU on its own EnergyNet and exposes Forge Energy conversion through its FE-aware blocks. The only unit both mods always agree on is FE — so FE is the bridge, and the patch just makes both sides convert at the same rate.

mekanism

Joules ⇄ FE

Native. Universal Cables, Energy Cubes, and every machine port take FE directly at JoulePerForgeEnergy (2.5 J / FE).

ic2 classic

EU ⇄ FE

Via FE-aware blocks at fluxBalance (4 FE / EU). This is IC2C's doorway onto any Forge-Energy grid.

the catch

One rate, both sides

If the two ratios disagree, every EU→FE→EU round-trip quietly multiplies or destroys power. The patch pins them so the loop is lossless.

No coremod required

On 1.19.2 this interop is entirely config-driven — there is nothing to compile or inject. The "patch" is a small, idempotent edit to two config files plus the in-game wiring described below.

01 — The numbers

The exchange rate

Three config values set the whole bridge. The trick is that they must satisfy one equation, or the conversion leaks:

JoulePerEU ÷ JoulePerForgeEnergy = fluxBalance 10 ÷ 2.5 = 4 ✔

With that satisfied, Mekanism's internal view (1 EU = 10 J = 4 FE) and IC2 Classic's view (1 EU = 4 FE) line up exactly:

Why it round-trips losslessly

1 EU → ×4 → 4 FE → ×2.5 → 10 J, and back the same way. These are also the stock defaults for both mods, so the pack was already aligned — the patch's job is to lock and enforce that across everyone's configs and make the cross-mod path deterministic.

02 — The edit

The config patch

Mekanism — config/Mekanism/general.toml, section [general.energy_conversion]:

[general.energy_conversion] blacklistIC2 = true # force all cross-mod energy through the FE bridge JoulePerEU = "10" # 1 EU = 10 J blacklistForge = false # keep FE interop ON — this IS the bridge JoulePerForgeEnergy = "2.5000" # 1 FE = 2.5 J (=> 1 EU = 4 FE) blacklistFluxNetworks = false # allow Flux Networks' high-throughput FE

IC2 Classic — config/ic2c/ic2.cfg:

[balance.blocks] I:fluxBalance = 4 # RF per EU — 1 EU = 4 FE [plugins] B:mekanism = true # IC2C's Mekanism cross-mod plugin
Why blacklistIC2 = true

Leaving Mekanism's direct IC2-EU integration on means there are two ways for power to cross — Mekanism pretending to be an EU tile and the FE bridge — which can double-handle or mis-meter energy. Standardising on the single FE boundary keeps behaviour predictable. It's reversible: set blacklistIC2 = false (or run the patcher with --allow-direct-eu) if you'd rather let Mekanism read EU directly too.

Left untouched on purpose

The patch does not change energyEasyMode (the pack ships it false). With easy mode off, over-volting an IC2 block doesn't blow it up — the energy simply won't flow. That's the safer default; just bridge at the right tier (next section).

03 — In the world

Wiring it in-game

The config sets the rate; you still place a block at the boundary that does the converting. The pattern is short:

  1. Mekanism side — nothing special. Universal Cables, Energy Cubes, and machine energy ports all accept and emit FE directly. Treat the boundary as "a Forge-Energy grid."
  2. IC2 Classic side — convert at an FE-aware block. Use IC2C's FE-capable interface/transformer (confirm the exact block in your build via JEI & its tooltip). Push FE into it to make EU; pull FE out of it to drain an EU grid.
  3. Match the IC2 voltage tier to your FE throughput. IC2 is tiered and, with easy mode off, an over-voltage feed just stops flowing rather than exploding. Bridge at the right tier (below), throttling the Mekanism cube/cable output or stepping IC2 voltage with transformers.

Verify in-world

Put a known load on one side and read the other with an IC2 EU-Reader / Mekanism Network Reader (or just watch a battery fill). At a correct setup, EU×4 in = FE out, with no drift over time.

04 — Gotchas

Pitfalls & tuning

  • Over-volting stalls, it doesn't explode. With energyEasyMode=false, feeding an IC2 block above its tier means energy won't flow. If a bridged machine sits dead, you're probably pushing FE that maps above its EU tier — step it down or bridge higher.
  • Don't run both integration paths. If you set blacklistIC2=false and wire an FE converter, the same energy can cross two ways. Pick one — the patch picks FE.
  • Flux Networks rides the same FE. It's in this pack; blacklistFluxNetworks=false keeps its higher-throughput Forge-Energy path usable across the bridge. Wireless FE in, EU out works the same way.
  • These are world-restart, server-side options. Change them with the server stopped, and ship the patched files to everyone — they're authoritative on the server.
  • If a value looks off, trust the game. Should a future Mekanism/IC2C update rename a key or change a default, re-check the one equation — JoulePerEU ÷ JoulePerForgeEnergy = fluxBalance — against in-game tooltips and JEI.

05 — Ship it

Apply & verify

The repo ships an idempotent patcher (mekanism-ic2c-compat/apply_patch.py) that edits both files to the locked values and backs each up first. From the pack/server root:

# preview only — writes nothing python3 mekanism-ic2c-compat/apply_patch.py --config-dir config --dry-run # apply (timestamped .bak per file) python3 mekanism-ic2c-compat/apply_patch.py --config-dir config # keep Mekanism's direct EU path on as well python3 mekanism-ic2c-compat/apply_patch.py --config-dir config --allow-direct-eu
  1. Dry-run, then apply against the server's config/. The script is idempotent — re-running once values match is a no-op.
  2. Distribute the patched config/Mekanism/general.toml and config/ic2c/ic2.cfg to every member (server-authoritative options).
  3. Restart the server/world so the conversion rates load.
  4. Verify with a known load and a reader, as in §03.
Prefer editing by hand?

The exact patched blocks are committed at mekanism-ic2c-compat/reference/, and the full write-up lives in mekanism-ic2c-compat/README.md.
